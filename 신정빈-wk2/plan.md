# BIT Analysis Plan

## Goal
`BIT Analysis`를 취업 준비생을 위한 산업분석/기업분석 AI 리서치 플랫폼으로 만든다. 처음부터 실제 서비스로 발전 가능한 정보 구조, 데이터 연결 방식, 리포트 생성 흐름을 고려하고, 랜딩페이지와 웹앱형 작업 공간을 함께 갖춘 단일 `index.html` 결과물로 제작한다. 사용자는 산업명 또는 기업명을 입력하고, 최신 웹 리서치 흐름과 DART/OpenDART 기반 재무 분석 흐름을 통해 컨설팅식 리포트 초안을 얻는다.

## Audience
- 컨설팅, 전략기획, PM, 투자/리서치 직무를 준비하는 대학생 및 20대 취업 준비생
- 산업 스터디, 기업분석, 케이스 인터뷰, 자소서/면접 대비를 반복적으로 수행하는 사용자
- "어떤 자료를 봐야 하는지", "숫자에서 어떤 인사이트를 뽑아야 하는지", "리포트를 어떤 구조로 써야 하는지"가 막막한 사용자

## Product Positioning
한 줄 정의: `BIT Analysis`는 산업명 또는 기업명을 입력하면 최신 리서치 소스, DART 재무 데이터, 컨설팅식 분석 프레임을 결합해 취업 준비용 분석 리포트를 생성하는 웹 플랫폼이다.

핵심 포지셔닝:
- 일반 챗봇형 답변이 아니라, 리서치 절차와 산출물 포맷을 표준화하는 도구
- 산업분석과 기업분석을 모두 제공하되, 결과물은 면접/스터디/케이스 준비에 바로 쓸 수 있는 brief 형태
- 기업분석은 사용자의 기존 엑셀 템플릿처럼 `재무 데이터 입력 -> 지표/차트 생성 -> 인사이트 작성` 흐름을 따른다.
- 최신 웹 리서치와 DART 재무 데이터는 "source-backed analysis"의 근거로 표시한다.

## Product Scope
이번 결과물은 단일 HTML 기반이지만, 향후 실제 API와 백엔드 연결을 전제로 한 제품 구조까지 구현한다.

1. 완성도 있는 랜딩페이지
   - 첫 화면에서 브랜드, 문제, 가치 제안, 대표 사용 사례를 명확히 전달한다.
   - 단, 과장된 마케팅 페이지가 아니라 곧바로 제품 체험으로 이어지는 구조로 만든다.

2. 웹앱 작업 공간
   - `산업분석`과 `기업분석`을 탭으로 전환한다.
   - 입력 패널, 데이터 소스 패널, 리포트 미리보기, 지표/차트 영역을 포함한다.
   - 사용자가 입력한 산업/기업/목적에 따라 결과 텍스트와 리포트 구조가 바뀐다.

3. 최신 웹 리서치 흐름
   - 정적 HTML의 한계를 고려해 실제 검색 API 호출 대신 `Research Queue`를 구현한다.
   - 사용자가 입력한 주제에 대해 자동으로 검색 쿼리, 확인해야 할 출처 카테고리, 검증 질문을 생성한다.
   - 추후 Tavily, SerpAPI, Perplexity API, OpenAI web search, 또는 자체 백엔드 검색 프록시에 연결 가능한 구조로 JS 함수를 분리한다.
   - UI에는 `Live research pipeline` 상태와 소스별 신뢰도/사용 목적을 보여준다.

4. DART/OpenDART 연동 흐름
   - OpenDART는 원문 공시 XML, 주요 재무 계정, 기업 간 재무 계정 비교, XBRL 재무제표 다운로드를 제공한다.
   - 정적 HTML에서는 API 키 보안과 CORS 제약이 있으므로 두 경로를 제공한다.
   - `Local DART Upload`: 사용자가 DART/OpenDART에서 내려받은 XML/CSV 파일을 업로드하면 브라우저에서 일부 항목을 파싱한다.
   - `OpenDART Connector`: API 키, corp code, 사업연도, 보고서 유형을 입력하는 연결 패널을 제공하고, 서비스에서 호출할 엔드포인트/파라미터 구조를 시각화한다.
   - 기업분석 결과 패널은 매출, EBIT, EBITDA, 순이익, 자산/부채/자본, 총차입금, 현금성자산, 순차입금, 현금흐름, PER, EV/EBITDA를 중심으로 구성한다.

5. 리포트 생성
   - 산업분석 결과: 시장 정의, 시장 구조, 밸류체인, 주요 플레이어, 성장 동인, 리스크, why now, 케이스 인터뷰 질문
   - 기업분석 결과: 회사 개요, 사업 구조, 5개년 재무 하이라이트, 수익성/성장성/안정성 진단, 투자 포인트, 리스크, 면접용 talking points
   - Markdown 복사 기능을 구현한다.

## Content Sections
1. Landing Hero
   - 브랜드명 `BIT Analysis`
   - 핵심 메시지: "산업과 기업을 컨설팅식으로 읽는 가장 빠른 방법"
   - CTA: `산업분석 시작`, `기업분석 시작`
   - 첫 화면 안에 제품 UI의 일부가 보이도록 구성한다.

2. Problem / Use Cases
   - 산업스터디 시작이 막막한 상황
   - 기업분석 템플릿은 있지만 숫자 해석이 어려운 상황
   - 면접에서 "왜 이 산업/회사인가"를 구조적으로 말해야 하는 상황

3. Workflow
   - Step 1: 분석 대상 입력
   - Step 2: 웹 리서치와 DART 데이터 소스 수집
   - Step 3: 컨설팅식 프레임으로 구조화
   - Step 4: 리포트/면접 질문/인사이트 출력

4. Product Workspace
   - 산업분석 탭
   - 기업분석 탭
   - 데이터 소스 패널
   - 분석 결과 패널
   - 지표/차트 패널

5. Output Examples
   - `Industry Brief`
   - `Company Brief`
   - `Interview Talking Points`
   - `Risk Register`

6. Output Sample
   - 산업분석 산출물 예시
   - 기업분석 산출물 예시
   - 사용자가 기대할 수 있는 최종 결과 중심으로 설명

## Visual Direction
- SaaS + 리서치 대시보드 느낌의 차분하고 실무적인 디자인
- 기존 BIT 강의자료의 민트 포인트를 일부 계승하되, 전체가 민트 일색이 되지 않도록 차콜, 화이트, 잉크 블루, 연한 그레이를 함께 사용한다.
- 랜딩페이지는 고급스럽게 만들되, 첫 화면에서 제품이 무엇인지 바로 보이게 한다.
- 작업 공간은 카드 중첩을 피하고, 명확한 패널/섹션/테이블/차트로 구성한다.
- 데이터 도구답게 숫자, 라벨, 소스 상태, 리포트 문장이 모두 잘 읽히도록 한다.
- 모바일에서는 랜딩 -> 탭 -> 입력 -> 결과 순서로 자연스럽게 쌓이도록 한다.

## Interaction Plan
- Hero CTA 클릭 시 로딩 화면을 거쳐 웹앱 영역으로 전환
- 산업분석/기업분석 탭 전환
- 입력값 기반 리포트 제목과 본문 자동 업데이트
- `Generate Brief` 버튼으로 샘플 리포트 생성
- `Build Research Queue` 버튼으로 검색 쿼리와 소스 체크리스트 생성
- `Upload DART File`로 XML/CSV/TXT 파일을 읽고 파일명, 크기, 감지된 키워드, 일부 재무 항목 후보를 표시
- `OpenDART Connector`에서 OpenDART API 연결에 필요한 corp code, year, report code, statement type 구조를 사용자 친화적으로 보여줌
- `Copy Markdown` 버튼으로 결과 복사

## Implementation
- 생성 파일: `index.html`
- 형식: HTML, CSS, JavaScript가 모두 포함된 단일 파일
- 외부 빌드 없이 브라우저에서 열 수 있게 만든다.
- 정적 HTML에서 가능한 기능:
  - 랜딩페이지
  - 인터랙티브 분석 탭
  - 샘플 리포트 생성
  - 연구 쿼리/소스 체크리스트 생성
  - 로컬 DART 파일 업로드 및 텍스트/XML/CSV 일부 파싱
  - 간단한 차트/지표 시각화
  - Markdown 복사
- 추후 백엔드/API가 필요한 기능:
  - 실제 최신 웹 검색 결과 수집
  - OpenDART API 키 기반 실시간 호출
  - DART XBRL/ZIP/XLSX 완전 파싱
  - 기존 기업분석 엑셀 템플릿으로 `.xlsx` 자동 생성
  - AI 모델 기반 원문 요약 및 출처 인용

## Data / API Notes
- OpenDART 공식 문서 기준, DART 원문 공시와 문서는 Open API로 접근 가능하며 XML 형식 원문을 다운로드해 원하는 정보를 추출할 수 있다.
- OpenDART는 주요 재무 계정, 기업 간 계정 비교, XBRL 재무제표 다운로드 등 재무정보 관련 서비스를 제공한다.
- 정적 HTML에서 API 키를 직접 노출하면 보안상 부적절하므로, 실제 서비스에서는 서버리스 함수 또는 백엔드 프록시를 둔다.
- 이번 HTML에서는 이 제약을 숨기지 않고, 로컬 파일 분석과 실시간 DART 연동 준비 상태를 사용자 흐름 안에서 분리해 보여준다.

## Assumptions
- 이번 산출물은 과제 제출 및 공유용이면서, 이후 실제 서비스 개발로 확장할 수 있는 프론트엔드 기반 제품을 만든다.
- 실제 최신 웹 검색과 OpenDART 실시간 호출은 단일 HTML 안에서 보안/브라우저 제약이 있는 부분을 분리하고, 로컬 파일 업로드/샘플 데이터/연결 구조를 통해 개발 방향성을 명확히 보여준다.
- 브랜드명은 `BIT Analysis`로 확정한다.
- 사이트 언어는 한국어를 기본으로 하되, 리포트 섹션명 일부는 컨설팅/리서치 톤에 맞춰 영어 병기를 허용한다.
- 사용자가 승인하면 이 계획을 바탕으로 `index.html` 제작에 들어간다.
